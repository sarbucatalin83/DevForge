# Feature 1 — Learning Tracks

## Description

Three independent learning tracks: **JavaScript**, **TypeScript**, **React**. Each track
covers every topic from trivial/beginner through advanced/senior level. Content is
AI-generated by Claude via an offline CLI script, designed to leave no gap in the curriculum.

## Behaviour

- **Free navigation**: users can jump to any topic or difficulty at any time — no forced
  linear progression.
- **Sidebar filter**: filter by track, topic, and difficulty level:
  - Beginner
  - Intermediate
  - Advanced
  - Senior

## Extensibility

The content schema and executor configuration are language-agnostic. Adding a new track
(Go, Rust, etc.) requires only:
1. A new content bundle (JSON files) for that language.
2. A new entry in the Docker executor config file.

Zero frontend code changes are required to support a new language track.

## Track Structure (per track)

Each track is subdivided into **topics** (e.g. JavaScript → Closures, Promises, Prototypes).
Each topic has content at all four difficulty levels. Topics are defined by the content
generation script and stored in the content directory structure:

```
content/
└── <track>/
    └── <topic>/
        └── <difficulty>-<type>-<timestamp>.json
```

## Error States

- If no content exists for a selected filter combination, show a clear empty state with
  instructions on how to generate content using the CLI script.
