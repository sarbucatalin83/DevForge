# UI / UX Requirements

## Component Library

shadcn/ui throughout. No other component libraries or CSS frameworks are permitted.
Tailwind CSS is used exclusively via shadcn.

## Theme

- Dark / light mode toggle available in the top navigation bar.
- Default: system preference.

## Layout

### Quiz Mode
- Centered single-column card layout.
- Question card occupies the main content area.
- Answer text area directly below the question.
- Submit and Show Answer buttons at the bottom of the card.
- After submission: correct answer and explanation expand below the user's answer.
- Pass / fail self-assessment buttons replace Submit after submission.

### Code Editor Mode
- Two-panel horizontal layout:
  - **Left panel** (40% width): task description, difficulty badge, exercise type label.
  - **Right panel** (60% width): Monaco editor.
- Test results drawer or collapsible section below the editor.
- On failure: solution diff shown below the test results.

## Navigation

- **Top-level**: sidebar (collapsible) with the three track entries (JS, TS, React) and a
  Content coverage page link.
- **Secondary**: filter panel within each track — filter by topic and difficulty.
- **Mode toggle**: within a track, a tab or toggle switches between Quiz mode and Code
  Editor mode.

## Responsive

- Minimum supported viewport: 1280 px wide.
- Mobile and tablet layouts are out of scope for v1.

## Error States

All errors must be surfaced inline with human-readable messages and actionable guidance:

| Condition | Message shown |
|-----------|--------------|
| Docker not running | "Docker is not running. Start Docker Desktop and reload." |
| No content for filter | "No content found for this combination. Run: `node scripts/generate-content.js --track ... --topic ... --difficulty ...`" |
| Container timeout | "Time limit exceeded (10 s). Your code may have an infinite loop." |
| Missing Docker image | "Image `<name>` not found. Run: `docker pull <name>`" |

## Accessibility

- All interactive elements keyboard-navigable.
- Focus management after quiz submission (focus moves to self-assessment buttons).
- Monaco editor retains focus after submit while results load.
